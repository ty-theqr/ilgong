기술적 구현 아키텍처 (Hybrid 방식)

1. 통신사 플랫폼 역할 (Trigger)

복잡한 기능 없이 '단순 문자 발송' 기능만 사용합니다.

  • 1번 선택 시: "계좌번호 텍스트" 문자 발송
  • 2번 선택 시: "배송/반품조회 URL 링크" 문자 발송

2. 더큐알 자체 제작 (Landing Page)

고객이 링크 클릭 시 보여질 '모바일 웹페이지'를 직접 제작합니다. (홈페이지팀 2명 활용)

  • 배송조회: 스마트스토어/택배사 조회 페이지 링크 연결
  • 파손접수: HTML Form 제작 (사진 업로드 → 내부 서버 저장)

기능별 상세 구현 방안

  • 1

    계좌번호 안내 (Text Only)

    통신사 기본 기능 100% 활용

    ARS 설정: "1번 누르면 지정된 문구(계좌번호) SMS 발송"
  • 2

    배송 조회 (Link Redirect)

    API 연동 개발 비용 절감

    ARS 설정: "2번 누르면 [배송조회 바로가기 URL] 발송"
    URL 내용: 네이버페이 구매내역 or 택배사 조회 링크로 이동
  • 3

    파손 접수 (Mobile Web Form)

    더큐알 웹팀 자체 제작 페이지

    ARS 설정: "3번 누르면 [접수 페이지 URL] 발송"
    페이지 기능: 이름/연락처 입력 + 사진 첨부(input type="file")

실행 타임라인 (내부 제작 포함)

Step 1. 통신사 계약 (1주차)

"보이는 ARS" 또는 "콜백 문자 서비스" 가입. (월 3~5만원)

*KT/LG 상담 시 "버튼 누르면 문자 나가는 기능" 확인

Step 2. 웹페이지 제작 (1주차)

홈페이지팀: 모바일용 랜딩페이지 1장 제작.

  • 섹션1: 배송조회 링크 모음
  • 섹션2: 파손 사진 업로드 폼
Step 3. 연결 및 오픈 (2주차)

통신사 ARS 설정 페이지에서 2번, 3번 버튼에 제작한 URL 입력.
