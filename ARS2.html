상담형 주문 최적화 프로세스 (Human + Tech)

1. 전화 상담 (기존 유지)

"사장님, 오늘 굴 싱싱해요?"
"네, 2kg에 5만원입니다."

2. 결제요청서 발송 (클릭)

상담원이 PC에서 금액(5만원) 입력 후 [문자발송] 버튼 클릭

3. 고객 결제 & 주소입력

문자 링크 클릭 → 금액확인
→ 계좌이체/카드결제 → 주소 직접 입력

이 방식의 결정적 장점

상담원이 계좌번호를 불러주거나 문자로 칠 필요가 없습니다.
가장 중요한 '배송 주소'를 고객이 직접 입력하게 하여 오배송 사고를 원천 차단합니다.
(어르신들도 '금액 확인' 화면에서 주소 넣는 방식은 홈쇼핑 ARS보다 훨씬 쉬워하십니다.)

추천 플랫폼 (SMS 결제요청)

  • A

    페이앱 (PayApp) / 블로그페이

    국내 1위 수기결제/링크결제 솔루션.

    • 상담원 PC 관리자 페이지 제공
    • 카드결제, 휴대폰결제, 무통장입금 모두 지원
    • 주소 요청 기능 기본 탑재
  • B

    PG사 수기결제 (나이스페이, 토스 등)

    기존 PG사를 쓰고 계시다면 'SMS 결제' 부가서비스 신청.

    *단, '주소 입력 기능'이 약할 수 있어 확인 필요.

실행 로드맵 (Action Plan)

Step 1. 솔루션 가입 (즉시)

'페이앱' 등 SMS 결제 서비스 가입 (가입비 약 10~20만원, 연회비 없음)

Step 2. 상담원 교육

"고객님, 금액과 계좌번호가 적힌 결제창을 문자로 보내드릴테니 주소 입력해주세요~" 멘트 교육.

Step 3. 보이는 ARS 연동 (선택)

배송조회/단순문의는 여전히 '보이는 ARS'로 방어하고,
주문상담만 상담원에게 연결되도록 설정.

CS 처리 시간 단축 예상효과

Before (기존)

상담(2분) + 계좌안내/받아적기(3분) = 총 5분 소요

After (개선)

상담(2분) + 링크전송(10초) = 총 2분 10초 소요
